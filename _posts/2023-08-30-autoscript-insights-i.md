在 **AutoScript** 中，**实体（Entity）** 是一等公民。所有可以绑定为变量的都被称为实体，其包括我们熟悉的*对象（Object）*、*函数（Function）*、*类型（Type）*。这些本应不相容的概念在 AutoScript 是所有实体共有的特征。本节让我们探索实体的行为特征。

## 变量

**变量（Variable）** 是绑定了名字的实体，在拥有名字之后我们便可以重复引用同一个实体了。这个绑定的过程需要通过变量定义完成。

### 变量的定义

AutoScript 中，变量定义以一个*未定义*的标识符开始，此时编译器便知道这是一个定义语句。随后，使用 `=` 来引出标识符绑定的对象。最后，作为一个语句的结尾，我们加上分隔符 `;` 告诉编译器变量定义完成。

```autoscript
x = 42;        // 定义了一个变量，它是名字 x 对对象 42 的绑定
```

这种定义方式和 Python 等脚本语言颇为相似，但笔者并非单纯是为了简洁性而作出这样设计的，下面让我们对比不同语言的变量定义语法进行分析。

#### 利用初始化器的定义

在 C/C++ 中，变量定义是由一个类型限定符、一个声明器和一个初始化器组成的，比如：

```c
int arr[] = { 1, 2, 3 };
void *ptr = NULL;
```

这样设计的好处在于显式类型指定有助于读者阅读代码，可惜初始化器堪称灾难的设计让这个优点并不显著。这里的缺点则同样也是不必要的类型声明：如果编译器可以通过等号右侧推断出类型，那么类型标注完全是冗余的。

#### 利用关键字的定义

在许多其它语言（包括 C++11 后的 C++）中，我们可以用一个关键字来提示编译器这是一个变量定义，比如 `let` 或 `var` 等。以 Rust 为例：

```rust
let x = 42;
let mut y = 24;
```

这充分利用了编译器强大的类型推断能力。不过它还是有值得改进的地方：在定义没有出现过的变量时，编译器自动就能判断出这是“理应被定义”的变量，那么 `let` 这样的关键字就没有意义。此外，`let mut` 这样的声明方式将实体上的性质限制在了变量上，我们就缺少直接在实体上修饰的方式（比如我们应该怎样写出一个拥有静态存储期的字面量呢？），更理想的方式可能是在等号右侧使用修饰符。

#### 直接定义

Python 等脚本语言中不需要任何准备，直接用一个等号就可以定义变量：

```python
x = 42
x = None
```

不过这也带来一个很显然的问题，就是无法区分定义和赋值。这在一些情况下会让 debug 变得比较艰难。此外，Python 为了防止不同作用域之间的名字混淆，还引入了 `global` 和 `nonlocal` 关键字，为语言增加了一些噪音。

#### AutoScript 的方案

AutoScript 结合关键字和直接定义的两套方案，并做出了改进。首先，分离变量定义和赋值两个操作。这两者在逻辑上本来就有许多不同，如果使用一个符号容易造成混淆。这样就不用担心定义和声明之间的歧义了。同时，关键字也可选使用。

```autoscript
x = 42;
var y = 24;
```

在后续小节和文章中我们会看到，`var` 是和 `sym`、`alias`、`macro` 并列的，用于修饰其后名字含义的关键字，因此这里保留关键字 `var` 也是一种统一性。

`var` 在一些情况下是必须的，即如果我们在外部作用域已经定义了同名的变量，此时如果想要 *覆盖* 它，就必须使用 `var` 关键字。

```autoscript
x = 42;
main = routine {
    x = 24;       // 编译错误，x 已经被定义
    var x = 24;   // 没问题，覆盖了全局空间的 x
};
```

因此，AutoScript 鼓励使用 `var`，习惯上只有全局变量才会省略 `var`。

### 类型推断

AutoScript 中有很多内置类型，它们是通过 *字面量（Literal）* 的方式进入程序当中的。在变量定义时，编译器会分析等号右侧对象的类型，并判断是否存在错误。在 AutoScript 中，我们可以在变量定义时指定其类型。这是通过类型声明器做到的。

```autoscript
var x : Int = 42;
```

此时会将 `x` 的类型设置为整数类型 `Int`。注意到这样的设计给类型转换发挥的空间。等号右侧的对象可以以和显式声明的类型不同。

```autoscript
var x : Real = 0;
```

这里 `0` 会被**隐式转换（Implicit Conversion）** 为 `Real` 类型，得到的对象会绑定到 `x` 上。

### 提前声明

变量的名字可以通过提前声明的方式提前引入到程序中。这几乎是 C/C++ 特有的语言特性。AutoScript 中的提前声明有一个非常简洁的语法，即 `name : type`。和 `var` 类似地，我们也可以使用关键字 `sym` 来提示 `name` 是一个提前声明。和 `var` 几乎一致地，如果外部作用域中已经定义了同名变量，我们必须在内部作用域使用 `sym` 来覆盖这个名字。

```autoscript
sym x : Int;      // 声明 x 是一个整数
var x = 42;       // x 的定义
```

提前声明可以出现多次，但定义只能出现一次。这和 C/C++ 的 *唯一定义原则（One Definition Rule, ODR）* 是同理的。

## 对象

**对象（Object）** 描述了实体的数据性。在大多数情况下我们看到的变量，其都作为对象在程序中发挥作用。对象主要有 *存储期*、*可变性*、*原子性* 和 *动态性* 四个特征。在仔细探索这几个特征之前，我们有必要先了解和对象相关的两个重要概念，*子对象* 和 *间接对象*。

### 子对象

一个对象可以由零到多个 **子对象（Subobject）** 组成。通常，对象的性质会被子对象自动继承，但子对象可以主动选择和父对象不同的性质。其它语言中的子对象常见于数组、元组或结构类型中：

```cpp
int arr[3] = {};    // 包含 3 个子对象的数组
std::tuple<int, double> tup = { 42, 1.0 };  // 包含 2 个子对象的元组
struct { int a, b; } s = { 1, 2 };          // 包含 2 个子对象的结构对象。
```

AutoScript 中常见的子对象出现于 *元组（Tuple）* 中。

```autoscript
tup = (1, 2);    // 元组对象 (1, 2) 由两个子对象 1 和 2 组成
```

子对象总是拥有相邻的内存地址，这和 C++ 中的子对象系统如出一辙。

### 间接对象

一个对象中可能并不直接存储具体信息，它可能需要经过一层 *解引用* 才能获得实际数据。这样的对象被称为 **间接对象（Indirectional Object）**。比如 C++ 中的指针：

```cpp
int x = 42;
int* p = &x;        // p 是一个间接对象
printf("%d", *p);   // 经过解引用后才能得到 p “存放”的数据
```

C# 等语言中的大多数对象默认都是间接对象。

```csharp
MyClass x = new MyClass();  // x 是一个间接对象
```

Haskell 等语言中的数据类型也是间接对象。

```haskell
data Tree a = Leaf a | Branch a (Tree a) (Tree a)
```

AutoScript 中的引用类型和动态类型都有这样的特征。

```autoscript
x = 42;
r = ref x;      // 间接对象 r，其间接引用了对象 x
d = dyn x;      // 间接对象 d，其间接引用了 x 的一个复制
```

### 可变性

**可变性（Mutability）** 描述对象是否能被修改。下面让我们考察不同语言中可变性的设计。首先值得一提的是，大多数语言中将可变性设置为*变量*而非对象的性质，因此下面我在措辞上会更多地以“变量”而非对象为主题。在 AutoScript 中，可变性依然是对象的性质；但由于变量总是一个名字对一个对象的唯一映射，因此使用“变量的可变性”也没有歧义。

#### 默认可变性

大多数主流编程语言中的变量都是默认可变的，我们在定义之后可以随时对其赋值。如果需要变量不可变则要额外使用关键字。

```cpp
int x = 42;
int const y = 42;   // 不可变
x = 24;
```

在部分语言中则默认变量不可变，这在一些情况下可以提升程序效率和安全性。如果需要变量可变则要额外使用关键字

```rust
let x = 42;
let mut y = 42;     // 可变
y = 24;
```

一些函数式语言中，所有变量一经定义不得赋值，甚至也没有对其赋值的方法，比如 Haskell：

```haskell
x :: Int
x = 42              -- x 定义之后就不可能再修改了
```

#### 可变传递性

变量的可变性会传递到子对象中。以 C++ 为例：

```cpp
struct foo {
    int x = 42;  
};

int main() {
    foo f1;
    foo const f2;
    f1.x = 24;     // 没问题
    f2.x = 24;     // 编译错误
}
```

不过，间接对象的可变性*不一定*反映到其引用的对象上，以 JavaScript 为例：

```javascript
const arr = [1, 2, 3];  // arr 是一个不可变的间接对象（JS 中没有结构类型因此这和 C++ 中的数组不同）
arr[0] = 3;             // 引用对象的子对象可变
```

#### 可变退化

注意到变量的可变性不一定在定义变量时作为类型信息传递给新的变量，它甚至不一定出现在类型系统中（比如 JavaScript）。以 C++ 为例：

```cpp
auto const x = 42;     // 不可变变量
auto y = x;            // y 的类型是 int，x 的不可变性并没有继承到等式左侧
auto p = &x;           // p 的类型是 int const*，注意这里的 const 保留了下来
auto q = *p;           // q 的类型是 int
```

这其中涉及到 C++ 复杂的值类别转换，但我们可以从中看出规律：最外层的可变性修饰会在对象绑定时被忽略。这有一个明显的好处：它落实了语言中默认的可变性，即每个变量的默认可变性总是确定的，而不受对象特征干扰。不过这也会为语言增加隐式行为，对语言一致性的观感上造成损害。

#### AutoScript 的方案

AutoScript 将可变性纳入对象的类型当中，且贯彻修饰符随对象一起出现的原则。同时，AutoScript 决定让对象默认不可变，因此我们会使用 `mut` 关键字来建立一个可变对象。

```autoscript
x = mut 42;         // 可变对象
y = x;              // 可变对象
z : Int = x;        // 不可变对象
```

等号左侧和右侧的类型总是相同的，除非有显式声明的类型（如上面的 `z`）。显然，`mut Int` 隐式类型转换为 `Int` 是非常合理的操作。

对于已经定义完毕的变量，其赋值操作会比定义宽松一些，比如 `mut Int` 可以通过 `Int` 赋值。

```autoscript
x = mut 42;
x <- 24;
```

可变性会继承到子对象上，但不会继承到间接引用对象上，以 AutoScript 中的元组和引用类型为例：

```autoscript
x = mut (1, 2);     // 子对象的类型都是可变整数类型
x[1] <- 42;
r = ref x;          // 间接对象的类型是引用可变整数类型，但引用本身是不可变的
mr = mut ref x;     // 间接对象的类型是可变引用
```

不过一些同时是间接对象且拥有子对象的实体，我们依然会传递可变性。这里以动态对象和选择对象为例（后面会再提到）：

```autoscript
x = 42;
d = mut dyn x;         // 这里 d 和其引用的对象都是可变的
c : Int | String = 42;
e = mut c;             // 这里的 e 和其引用的对象都是可变的
```

换种角度来说，除了引用之外，AutoScript 总是会传递可变性。

### 原子性

对象的 **原子性（Atomicity）** 描述了其进行读取时的特征。一个原子的对象保证所有赋值操作都是一个机器指令，因此进行读写操作时不会出现数据竞态。大多数语言的院子类型都是通过内置的特殊类型做到的，尽管在库层面会有封装，比如 C++ 中的 `std::atomic<T>` 模版。然而，显然不是所有的类型都能通过机器指令做到单指令赋值，因此不支持的原子类型会通过 *互斥锁（Mutex）* 的方式实现原子操作。AutoScript 将对象的原子性纳入类型系统中，从而让对象的行为更加易于预测。原子对象通过 `atom` 修饰符创建，它和 `mut` 之间可以互换顺序而不改变含义。

```autoscript
x = atom mut 42;
x <- 24;             // 赋值操作保证是原子的
```

和可变对象类似地，赋值时编译器不会在意最外层的原子性是否一致。

```autoscript
x = atom mut 42;
x <- 24;
y = mut 42;
y <- x;
```

原子性也可以传递到子对象上。

```autoscript
tup = atom (1, 2);   // 和可变性类似，原子性会传递到子对象上
tup[0] <- 42;        // 原子的赋值操作
```

不过原子性不会传递到 `ref` 修饰的对象上，道理和 `mut` 完全一致。

```autoscript
x = 42;
a = atom ref x;
a <- 24;             // 并不是原子操作
b = atom dyn x;
b <- 24;             // 原子操作
```

### 动态性

**动态性（Dynamicity）** 为对象赋予动态运行时的性质。这在大多数静态语言中都是匮乏的功能（尽管我们可以构建虚拟机实现）。JavaScript 可以轻松地为对象添加成员：

```javascript
let x = 42;
x["entry"] = "Powerful!";
console.log(x.entry);     // 输出 Powerful!
```

上面的变量 `x` 定义时拥有 `number` 类型，但它却可以拥有额外的成员。AutoScript 也可以通过声明动态对象，在不打破类型系统的同时添加更多动态的信息。

```autoscript
x = dyn 42;             // 动态对象是一个间接对象，其指向的对象拥有一个对象头，可以为其添加额外信息
new x.entry = "Cool!";  // 在 x 上动态创建了一个成员 entry
print x.entry;          // 输出 Cool!
```

动态性有一个 **自动折叠（Automatic Collapsing）**  的特点，即多个嵌套的 `dyn` 修饰符不会生效，即使上面使用了 `atom` 或 `mut` 修饰符。比如 `dyn atom dyn 42` 和 `atom dyn 42` 无异，但 `dyn ref dyn 42` 和 `ref dyn 42` 或 `dyn ref 42` 都不同。

进行赋值时，动态修饰符同样不重要。

```autoscript
x = dyn mut 42;
x <- 24;
y = mut 42;
y <- x;
```

和前面两种修饰符不同的是，对象的动态性不会传递到子对象或引用对象上。比如 `dyn (1, 2)` 和 `dyn (dyn 1, dyn 2)` 拥有不同的类型。实际上和其它的对象性质相比，动态性更加类似于 `ref`，其直接改变了对象的内存布局（即将其变成一个间接对象）。

### 存储期

对象的 **存储期（Storage Duration）** 描述了实体在程序中存在的周期。这是 C 引入的概念。

```c
int main() {
    int x = 42;             // 自动存储期
    static float f = 0.f;   // 静态存储期
    int* arr = malloc(3 * sizeof(int));  // arr 引用的对象是动态存储期，但 arr 本身是自动存储期
}
```

其它多数语言中并没有存储期的概念，比如 Python 中所有的对象都拥有动态存储期，且经 GC 托管。AutoScript 受 C++ 的影响，决定引入存储期的概念。它们同样构成类型的一部份，且分为下面几种：

* 将亡存储期（Expiring）：实体只存在于当前语句。所有未绑定到名字上的对象默认都拥有这样的存储期。编译器可以视情况对其优化。
* 自动存储期（Automatic）：实体只存在于当前作用域，用 `auto` 关键字表示，它是所有绑定到名字上的对象的默认存储期。
* 静态存储期（Static）：实体存在于程序运行期间，用 `static` 关键字表示。
* 线程存储期（Thread）：实体存在于整个线程运行期间，用 `thread` 关键字表示。
* 常量存储期（Constant）：实体存在于编译期和程序运行期间，用 `const` 关键字表示。
* 动态存储期（Dynamic）：实体的存在周期由程序员手动决定，用 `alloc` 关键字表示。

```autoscript
x = 42;           // x 绑定到一个自动存储期变量上
y = static 42;    // y 绑定到一个静态存储期变量上
z = x + 1;        // 这里的 1 拥有将亡存储期，但 x + 1 的结果对象拥有自动存储期
w = alloc "abc";  // w 绑定到一个动态存储期变量上
```

存储期构成类型的一部份，且其中的隐式类型转换是单向的：可以以 `alloc`、`const`、`static`、`thread`、`auto` 的顺序从左到右隐式转换，但反过来是禁止的。

```autoscript
x = static 42;
y : ref static = ref x;   // 将 ref auto Int 转换为 ref static Int，没问题
z : ref const = ref y;    // 编译错误，不能将 ref static Int 转换为 ref const Int
```

由于存储期和可变性类似，会在变量定义时自动匹配过来，因此在只需要自动存储期的情形通常会使用 `auto` 关键字。

```autoscript
x = static 42;
y = x;            // y 也拥有静态存储期
z = auto x;       // 相当于将 x 复制出来一份自动存储期的对象
w = auto z;       // 没问题
```

其它的存储期限定符也可以像 `auto` 一样，拥有复制的功能，在使用于带有其它存储期限定符的对象上时会自动覆盖它的存储期性质。

存储期会自动传递到子对象上，但对于间接对象，其指向对象的存储期和其本身无关。动态对象的指向对象 *总是* 动态存储期的；引用对象的指向对象虽然不受间接对象的存储修饰符修饰，但它必须比间接对象拥有更长的存储期。

```autoscript
x = static 42;
r = ref x;         // 没问题，r 的存储期（auto）比 x 的存储期短
y = 42;
s = static ref y;  // 编译错误， static ref auto Int 不是一个合法类型
t = static ref const y; // 没问题
```

## 类型

### 内存大小和内存布局

**类型（Type）** 是实体的静态标签。在 AutoScript 中，有两种重要的类型：**内存大小（Memory Size）** 和 **内存布局（Memory Layout）**。它们分别对应了对象的大小和对象的性质。我们可以用 `size` 关键字构建一个特定字节大小的类型。

```autoscript
x : size 8;         // 声明了一个内存大小为 8 字节的变量 x
```

内存布局则可以通过已经定义的类型搭配上一节中介绍的各种修饰符和运算符构建出来。

```autoscript
x : (ref size 8, Int);    // 声明了一个元组对象，其中包含一个大小为 8 的子元素和一个 Int 子元素
```

所有类型都是某个内存布局类型的 *子类型*，而内存布局类型也是某个内存大小类型的子类型。有关子类型我们会在下一篇文章中仔细讨论，不熟悉编程语言中这个概念的读者这里可以理解为，`A` 的子类型 `B` 总是可以当作 `A` 来使用。

对于其它类型，其本质是一个编译期 *约束（Constraint）*，即一个接受特定类型参数的编译期函数，其返回一个布尔类型。编译时，编译器会对尝试传入的参数类型交由这个约束来判断，若成功则构成一次函数调用。

```autoscript
Even = const funct (x : Int) -> x `mod` 2 == 0;
halve = const funct (x : Even) -> x / 2;
```

我们可以通过内置函数 `sizeof` 得到对象的类型：

```autoscript
x : size 8;
y : (ref size 8, Int);
print typeof(x);        // 输出 size 8
print typeof(y);        // 输出 (ref size 8, Int)
```

如果想要得到某个对象作为内存布局类型时的值，可以使用内置函数 `decay`。

```autoscript
EitherEven = const funct (x : Int, y : Int) -> x `mod` 2 == 0 or y `mod` 2 == 0;
halve_once = const funct (p : EitherEven) -> decay p;   // 得到 (Int, Int) 类型
```

另一个内置函数 `decayof` 可以得到内存布局类型。

下一节中让我们介绍一些更加复杂的类型。

### 结构和选择类型

#### 别名

**别名（Alias）** 是 *宏（Macro）* 系统中的一个特性，它可以在编译期将一个名字直接替换为另一个名字。我们可以用这种方式声明类型的别名。

```autoscript
macro Float = Real;
x : Float = 42;      // 等价于 x : Real = 42;
```

别名并不是一个名字绑定，因为它不会向程序引入新的 *符号*。编译得到的符号表中只会存在经 `var` 声明的名字。

#### 复制类型声明

我们可以将某个类型复制出来并定义新的类型。这个过程和变量定义完全一样。

```autoscript
var Float = Real;
```

这样创建的类型和原来的类型除了符号之外性质完全相同。同时，编译器会自动定义两种类型之间的隐式转换规则。这种方式比别名的好处在于，除非进行强制类型转换，否则两种类型之间是不相容的。旧类型的对象不能直接代入到新类型的环境中。

```autoscript
var Name = String;
x : Name = "abc";
y = x;               // 类型是 Name 而非 String
```

值得一提的是，上面声明新类型时并没有指定其存储期和可变性，此时会自动继承内置类型 `Int` 和 `String` 的相应性质，即静态且不可变。

#### 结构声明

**结构（Structure）** 是一个元组的模版，且其中的子对象都被赋予了一个名字。通过结构类型声明的对象也被称为 *具名元组（Named Tuple）*。结构类型是其对应匿名元组类型的子类型。

```autoscript
Pair = const struct (
    x : Int,
    y : Int
);
p : Pair 
```

每个子对象的声明都是用了提前声明的语法，因此上面的 `Pair` 也可以定义成：

```autoscript
Pair = const struct (
    sym x : Int,
    sym y : Int
);
```

不过习惯上并不需要这些关键字。

结构声明中的 `struct` 关键字并不是必须的，AutoScript 使用这样的“阐述关键字”是为了减轻编译器的编写负担。当我们已然知晓某个名字链接到一个类型上时，就不需要使用 `struct` 关键字了：

```autoscript
Pair : Type;
Pair = const (
    x : Int,
    y : Int
);
```

#### 抽象数据类型声明

**抽象数据类型（Abstract Data Type, ADT）** 是一种 *和类型（Sum Type）*。它拥有一系列标签和对应的类型。

```autoscript
Color = const data (
    RGB (r : Int, g : Int, b : Int) |
    NoColor
);
```

这里面出现的 `RGB` 和 `NoColor` 是类型 `Color` 的可选 *标签（Tag）*，它们代表了一个类型的动态性质。声明标签时，后面或者跟着一个类型，或者不接任何类型。标签声明之间通过 `|` 符号分隔。和类型不同的是，我们不能通过显式类型声明的方式创建一个标签类型的变量。

```autoscript
c : RGB = (0, 0, 0);    // 编译错误，RGB 不是一个类型
```

### 类型标注

实体拥有类型性质时，其最重要的特点是可以作为类型标注出现。所谓类型标注，就是类如变量声明时可选的显式类型声明那样，为一个名字声明其类型信息。只有 `const Type` 类型的对象才拥有这种性质。其余的所有 `Type` 类型都称为 **标签（Tag）**，这是一种动态存储的信息。

```autoscript
MyType = const struct ( // 声明了一个类型
    x : Int
);
MyTag = struct (        // 声明了一个标签
    x : Int
);
x : MyType = 42;
```

拥有类型和标签特性的实体都是 `Type` 类型的（毕竟他们拥有几乎相同的构建方式）。类型可以当作标签使用，但反过来是不行的。AutoScript 中的选择类型便是一种简化的 ADT（类似于元组相对于结构）：

```autoscript
IntOrString = Int | String;  // 效果类似于 data (Int Int | String String)
x : IntOrString = 42;        // x 的类型是 Int | String，但标签是 Int
```



## 函数

**函数（Function）** 描述了实体的变换方式，将一个函数 *应用（Apply）* 到符合条件的实体上时，会根据函数的定义得到特定的结果。函数的类型是 `T -> U` 的形式，其中 `T` 是其能接收参数的类型，而 `U` 是作为结果返回的类型。

### 函数声明

函数声明是一个模式声明和一个表达式的组合，两者通过箭头运算符 `->` 相连：

```autoscript
id = const funct (x : Int) -> x;
```

上面，`const` 是对函数整体的修饰，表示其拥有常量存储期。`funct` 和 `struct` 的作用类似，用来告知编译器这是一个函数实体。随后的 `(x : Int)` 构成一个模式声明，它会作为函数调用时对参数匹配的模式。箭头运算符后面的部分则描述了参数变换的表达式，经过函数调用后的参数会用来得到这个表达式的结果。

当我们已经声明了函数的 *类别（Kind）* 时，就不需要显式写出 `funct` 关键字了。AutoScript 中实体的类别指的是它 *连续* 经过多少调用后它才不能被继续调用。比如 `42` 的类别是 `0`。

```autoscript
sym add : Int -> Int -> Int;    // 类别为 2 的函数
var add = const (x : Int) -> const (y : Int) -> x + y;
```

值得一提的是，函数的每个参数都构成一个声明，我们可以用 `sym` 关键字作为提示词。

```autoscript
id = const funct (sym x : Int) -> x;
```
